# Week 6 Activity: Introduction to Prototyping and Wireframing
# 第6週アクティビティ：プロトタイピングとワイヤーフレーミング入門

## Activity Information | アクティビティ情報

**Activity Title:** Creating Digital Wireframes in Figma/Adobe XD | Figma/Adobe XDでデジタルワイヤーフレームを作成
**Duration:** 50-60 minutes | 50-60分
**Learning Objectives Addressed:** Create low to mid-fidelity wireframes using no-code prototyping tools | ノーコードプロトタイピングツールを使用して低～中忠実度のワイヤーフレームを作成する
**Prerequisites:** Week 5 completion - Paper sketches and mobile design principles understood | 第5週完了 - 紙のスケッチとモバイルデザイン原則が理解されている

---

## Activity Learning Objectives | アクティビティ学習目標

Students will:
学生は以下ができるようになります：

- Distinguish between wireframing and prototyping at different fidelity levels | 異なる忠実度レベルでワイヤーフレーミングとプロトタイピングを区別する
- Navigate Figma or Adobe XD interface and use basic tools effectively | FigmaまたはAdobe XDインターフェースをナビゲートし、基本的なツールを効果的に使用する
- Convert paper sketches into digital wireframes with proper spacing and alignment | 紙のスケッチを適切なスペーシングと整列を持つデジタルワイヤーフレームに変換する
- Apply mobile design grids and layout systems for consistency | 一貫性のためにモバイルデザイングリッドとレイアウトシステムを適用する

### Step 1: Tool Setup and Interface Familiarization (12 minutes) | ツールセットアップとインターフェース慣れ（12分）

**What Students Do:**
- Choose between Figma (recommended) or Adobe XD | Figma（推奨）またはAdobe XDから選択

- Complete bot's interactive tool tutorial | ボットのインタラクティブツールチュートリアルを完了

**Figma Setup (Recommended):**
1. Visit figma.com and create account | figma.comにアクセスしてアカウントを作成
2. Select "Education" plan if available (free) | 利用可能な場合「教育」プランを選択（無料）
3. Create new project: "HCI-101 Final Project" | 新しいプロジェクトを作成：「HCI-101最終プロジェクト」

**Adobe XD Setup (Alternative):**
1. Download Adobe XD from adobe.com/products/xd | adobe.com/products/xdからAdobe XDをダウンロード
2. Sign in with Adobe account (free plan available) | Adobeアカウントでサインイン（無料プラン利用可能）
3. Create new document: Mobile (iPhone 14 Pro: 393 x 852px) | 新しいドキュメントを作成：モバイル（iPhone 14 Pro：393 x 852px）
4. Save as: "HCI-101_Week6_Wireframes" | として保存：「HCI-101_Week6_Wireframes」

**Figma Essential Tools:**

- **Hand tool (H)** | ハンドツール（H）: Navigate canvas

- **Grids & Layouts** | グリッドとレイアウト: Enable alignment guides

**Adobe XD Essential Tools:**
- **Artboard tool (A)** | アートボードツール（A）: Create screens
- **Rectangle (R)** | 長方形（R）: Draw shapes

- **Padding** | パディング: Adjust spacing
- **Components** | コンポーネント: Reusable elements

**Interface Navigation Practice:**

- Practice zoom (Cmd/Ctrl + scroll or Cmd/Ctrl + 1/2) | ズームを練習（Cmd/Ctrl +スクロールまたはCmd/Ctrl + 1/2）

**Bot Guidance:**
- Provides video tutorials for each tool | 各ツールのビデオチュートリアルを提供
- Offers keyboard shortcut cheat sheet | キーボードショートカットチートシートを提供
- Interactive exercises: "Create a rectangle, add text, align them center" | インタラクティブ演習：「長方形を作成し、テキストを追加し、それらを中央に整列」

- Troubleshoots common issues: tool not working, elements not selecting | 一般的な問題をトラブルシューティング：ツールが機能しない、要素が選択されない

**Expected Output:** Figma/XD account setup, tool familiarization completed, practice exercises done
期待される出力：Figma/XDアカウントセットアップ、ツール慣れ完了、練習演習完了

**Time Allocation:** 12 minutes | 12分

### Step 2: Setting Up Mobile Design System Basics (10 minutes) | モバイルデザインシステム基礎のセットアップ（10分）

**What Students Do:**
- Establish consistent design system elements for wireframes | ワイヤーフレーム用の一貫したデザインシステム要素を確立
- Create reusable components and styles | 再利用可能なコンポーネントとスタイルを作成

**Design System Elements to Create:**

**1. Grid and Layout System** | グリッドとレイアウトシステム
- Set up 8px or 4px grid system | 8pxまたは4pxグリッドシステムをセットアップ
- Create layout grid for mobile screens: | モバイル画面用のレイアウトグリッドを作成：
  - Columns: 4-6 columns | カラム：4-6カラム
  - Gutter: 16px | ガター：16px
  - Margin: 16-24px on sides | マージン：側面に16-24px
  - **Figma:** Frame > Layout Grid | フレーム > レイアウトグリッド
  - **XD:** View > Show Layout Grid | View > Show Layout Grid

**2. Typography Styles** | タイポグラフィスタイル
Create text styles for consistency: | 一貫性のためのテキストスタイルを作成：
- **H1 - Screen Title:** 28-32px, Bold | H1 - 画面タイトル：28-32px、太字
- **H2 - Section Header:** 20-24px, Semibold | H2 - セクションヘッダー：20-24px、セミボールド
- **Body Text:** 16-18px, Regular | 本文テキスト：16-18px、通常
- **Button Text:** 16-18px, Semibold | ボタンテキスト：16-18px、セミボールド
- **Caption/Helper:** 12-14px, Regular | キャプション/ヘルパー：12-14px、通常
- Font family: SF Pro (iOS), Roboto (Android), or Inter (universal) | フォントファミリー：SF Pro（iOS）、Roboto（Android）、またはInter（ユニバーサル）

**3. Spacing System** | スペーシングシステム
Standardize spacing using multiples of 4 or 8: | 4または8の倍数を使用してスペーシングを標準化：
- **XS:** 4px - Between related items | XS：4px - 関連アイテム間
- **S:** 8px - Within components | S：8px - コンポーネント内
- **M:** 16px - Between components | M：16px - コンポーネント間
- **L:** 24px - Between sections | L：24px - セクション間
- **XL:** 32px - Major page sections | XL：32px - 主要ページセクション

**4. Basic UI Component Library** | 基本的なUIコンポーネントライブラリ
Create reusable wireframe components: | 再利用可能なワイヤーフレームコンポーネントを作成：

**Button Component:** | ボタンコンポーネント：
- Primary button: 327px wide × 48px tall (or full width - 32px margins) | プライマリボタン：327px幅 × 48px高（またはフルワイド - 32pxマージン）
- Secondary button: Same size, outlined version | セカンダリボタン：同じサイズ、アウトライン版
- Corner radius: 8-12px | コーナー半径：8-12px
- Text: Centered, 16-18px | テキスト：中央、16-18px

**Input Field Component:** | 入力フィールドコンポーネント：
- Width: Full width minus margins | 幅：フルワイドマイナスマージン
- Height: 48-56px | 高さ：48-56px
- Border: 1-2px solid | ボーダー：1-2px固定
- Label: Above or floating | ラベル：上またはフローティング
- Placeholder text: Gray | プレースホルダーテキスト：グレー

**Card Component:** | カードコンポーネント：
- Rounded corners: 12-16px | 丸いコーナー：12-16px
- Padding: 16px inside | パディング：内側16px
- Drop shadow: subtle (optional for wireframes) | ドロップシャドウ：微妙（ワイヤーフレームではオプション）
- Content: Image placeholder + text | コンテンツ：画像プレースホルダー+テキスト

**Navigation Components:** | ナビゲーションコンポーネント：
- Bottom tab bar: 56-64px tall, full width | 下部タブバー：56-64px高、フルワイド
- Tab items: Icons + labels, evenly distributed | タブアイテム：アイコン+ラベル、均等に分散
- Top navigation bar: 44-56px tall | トップナビゲーションバー：44-56px高
- Back button: 44x44px touch target | 戻るボタン：44x44pxタッチターゲット

**5. Color System for Wireframes** | ワイヤーフレーム用カラーシステム
Keep wireframes grayscale: | ワイヤーフレームをグレースケールに保つ：
- **Background:** #FFFFFF (white) | 背景：#FFFFFF（白）
- **Cards/Containers:** #F5F5F5 (light gray) | カード/コンテナ：#F5F5F5（ライトグレー）
- **Text:** #000000 or #333333 (dark gray) | テキスト：#000000または#333333（ダークグレー）
- **Borders:** #CCCCCC (medium gray) | ボーダー：#CCCCCC（ミディアムグレー）
- **Primary Action:** #007AFF or #666666 (blue or darker gray) | プライマリアクション：#007AFFまたは#666666（青またはダーカーグレー）
- **Placeholders:** #999999 (light gray) | プレースホルダー：#999999（ライトグレー）

**How to Create Components:**

**Figma:**
- Select elements | 要素を選択
- Right-click > "Create Component" (or Cmd/Ctrl+Alt+K) | 右クリック > 「コンポーネントを作成」（またはCmd/Ctrl+Alt+K）
- Name component: "Button/Primary" | コンポーネントに名前を付ける：「Button/Primary」
- Reuse by dragging from Assets panel | Assetsパネルからドラッグして再利用

**Adobe XD:**
- Select elements | 要素を選択
- Right-click > "Make Component" (or Cmd/Ctrl+K) | 右クリック > 「コンポーネントを作成」（またはCmd/Ctrl+K）
- Name in Layers panel | Layersパネルで名前を付ける
- Reuse by Alt/Option-dragging or from Components panel | Alt/Optionドラッグまたはコンポーネントパネルから再利用

**Bot Guidance:**
- Provides design system templates to import | インポートするデザインシステムテンプレートを提供
- Guides grid setup step-by-step | グリッドセットアップをステップバイステップでガイド
- Explains: "8px grid ensures consistent spacing throughout your design" | 説明：「8pxグリッドはデザイン全体で一貫したスペーシングを確保します」
- Helps create first components | 最初のコンポーネントの作成を支援
- Validates spacing and sizing against mobile guidelines | モバイルガイドラインに対してスペーシングとサイジングを検証

**Expected Output:** Mobile grid established, typography styles defined, basic component library created
期待される出力：モバイルグリッド確立、タイポグラフィスタイル定義、基本コンポーネントライブラリ作成

**Time Allocation:** 10 minutes | 10分

### Step 3: Converting Paper Sketches to Digital Wireframes (22 minutes) | 紙のスケッチからデジタルワイヤーフレームへの変換（22分）

**What Students Do:**
- Translate Week 5 paper sketches into digital wireframes | 第5週の紙のスケッチをデジタルワイヤーフレームに変換
- Create 4-6 key screens with proper alignment and spacing | 適切な整列とスペーシングで4-6個の主要画面を作成

**Wireframing Process:**

**For Each Screen (4-6 total):**

**Step A: Create Frame/Artboard** | フレーム/アートボードを作成
- Use Frame tool (Figma) or Artboard tool (XD) | フレームツール（Figma）またはアートボードツール（XD）を使用
- Choose mobile device preset: | モバイルデバイスプリセットを選択：
  - iPhone 14 Pro: 393 × 852px | iPhone 14 Pro：393 × 852px
  - iPhone 13/14: 390 × 844px | iPhone 13/14：390 × 844px
  - Android: 360 × 800px | Android：360 × 800px
- Name each frame clearly: "Home Screen", "Search Screen", etc. | 各フレームに明確に名前を付ける：「ホーム画面」、「検索画面」など
- Arrange frames in logical flow order (left to right) | 論理的なフロー順にフレームを配置（左から右）

**Step B: Add Layout Structure** | レイアウト構造を追加
- Enable layout grid (remember your 8px/4px system) | レイアウトグリッドを有効にする（8px/4pxシステムを覚えておく）
- Add main structural elements: | 主要な構造要素を追加：
  - **Status bar area** (top 44-50px) | ステータスバーエリア（上部44-50px）
  - **Navigation bar** (if applicable, 44-56px) | ナビゲーションバー（該当する場合、44-56px）
  - **Content area** (scrollable middle section) | コンテンツエリア（スクロール可能な中央セクション）
  - **Bottom tab bar** (if applicable, 56-64px) | 下部タブバー（該当する場合、56-64px）
  - **Safe area** (avoid notch and home indicator) | セーフエリア（ノッチとホームインジケーターを避ける）

**Step C: Add UI Components** | UIコンポーネントを追加
Using your component library or creating new elements: | コンポーネントライブラリを使用するか、新しい要素を作成：

**Navigation Elements:** | ナビゲーション要素：
- Top bar: Back button (44x44px), screen title, action buttons | トップバー：戻るボタン（44x44px）、画面タイトル、アクションボタン
- Bottom tabs: Icons (24x24px) + labels, 3-5 tabs maximum | 下部タブ：アイコン（24x24px）+ラベル、最大3-5タブ
- Hamburger menu if needed (44x44px) | 必要に応じてハンバーガーメニュー（44x44px）

**Content Elements:** | コンテンツ要素：
- Headings: Use text styles (H1, H2) | 見出し：テキストスタイル（H1、H2）を使用
- Body text: Use body text style | 本文テキスト：本文テキストスタイルを使用
- Lists: Cards or list items with consistent spacing | リスト：一貫したスペーシングを持つカードまたはリストアイテム
- Images: Gray rectangles as placeholders with "X" or icon | 画像：「X」またはアイコン付きプレースホルダーとしてグレーの長方形
- Icons: Simple shapes or text labels (24x24px) | アイコン：単純な形状またはテキストラベル（24x24px）

**Interactive Elements:** | インタラクティブ要素：
- Buttons: Use button components (primary/secondary) | ボタン：ボタンコンポーネント（プライマリ/セカンダリ）を使用
- Input fields: Use input components with labels | 入力フィールド：ラベル付き入力コンポーネントを使用
- Checkboxes/Radio buttons: 24x24px minimum | チェックボックス/ラジオボタン：最小24x24px
- Toggles/Switches: Standard platform size | トグル/スイッチ：標準プラットフォームサイズ

**Step D: Apply Consistent Spacing** | 一貫したスペーシングを適用
Follow your spacing system: | スペーシングシステムに従う：
- 16px margins from screen edges | 画面端から16pxマージン
- 16px spacing between components | コンポーネント間16pxスペーシング
- 24-32px spacing between sections | セクション間24-32pxスペーシング
- 8px padding inside components | コンポーネント内に8pxパディング
- Use alignment tools frequently | 整列ツールを頻繁に使用

**Step E: Add Annotations** | 注釈を追加
Help viewers understand functionality: | 視聴者が機能を理解するのを助ける：
- Use comment tool or text boxes | コメントツールまたはテキストボックスを使用
- Label interactions: "Tap to expand", "Swipe to delete" | インタラクションにラベルを付ける：「タップして展開」、「削除するにはスワイプ」
- Note states: "Selected state", "Error state" | 状態をメモ：「選択された状態」、「エラー状態」
- Explain non-obvious features | 明白でない機能を説明

**Wireframing Best Practices:**
- Keep it grayscale - no colors yet | グレースケールを保つ - まだ色なし
- Use real content where possible (not "Lorem ipsum") | 可能な場合は実際のコンテンツを使用（「Lorem ipsum」ではない）
- Show actual text length and line breaks | 実際のテキスト長と改行を表示
- Indicate scrollable areas | スクロール可能なエリアを示す
- Make touch targets clearly visible | タッチターゲットを明確に見えるようにする
- Maintain consistency across screens | 画面間で一貫性を維持
- Focus on layout and structure, not visual design | ビジュアルデザインではなく、レイアウトと構造に焦点を当てる

**Screens to Create (Priority Order):**
1. **Home/Dashboard Screen** - Main entry point | ホーム/ダッシュボード画面 - メインエントリーポイント
2. **Primary Task Screen** - Key user action | プライマリタスク画面 - 主要ユーザーアクション
3. **Content List/Browse** - Navigation and discovery | コンテンツリスト/閲覧 - ナビゲーションと発見
4. **Detail Screen** - Detailed information view | 詳細画面 - 詳細情報ビュー
5. **Form/Input Screen** - User data entry (if applicable) | フォーム/入力画面 - ユーザーデータ入力（該当する場合）
6. **Profile/Settings** - User account management | プロフィール/設定 - ユーザーアカウント管理

**Bot Guidance:**
- Provides wireframe templates for common screen types | 一般的な画面タイプのワイヤーフレームテンプレートを提供
- Reviews screens in real-time for common issues | 一般的な問題についてリアルタイムで画面をレビュー
- Checks: "Is that button 44px tall? Too small for touch target" | チェック：「そのボタンは44px高ですか？タッチターゲットには小さすぎます」
- Validates spacing: "Add 8px more spacing between these elements" | スペーシングを検証：「これらの要素間に8px以上のスペーシングを追加」
- Ensures consistency: "Button style is different on this screen - use your component" | 一貫性を確保：「この画面のボタンスタイルが異なります - コンポーネントを使用」
- Suggests improvements: "Consider adding a search icon in the navigation bar" | 改善を提案：「ナビゲーションバーに検索アイコンを追加することを検討」

**Expected Output:** 4-6 digital wireframes with consistent layout, proper spacing, and clear annotations
期待される出力：一貫したレイアウト、適切なスペーシング、明確な注釈を伴う4-6個のデジタルワイヤーフレーム

**Time Allocation:** 22 minutes | 22分

### Step 4: Consistency Check and Refinement (8 minutes) | 一貫性チェックと洗練（8分）

**What Students Do:**
- Review all wireframes for design consistency | すべてのワイヤーフレームをデザインの一貫性のためにレビュー
- Fix alignment, spacing, and sizing issues | 整列、スペーシング、サイジングの問題を修正
- Ensure mobile usability standards are met | モバイルユーザビリティ基準が満たされていることを確認

**Consistency Checklist:**

**1. Typography Consistency** | タイポグラフィの一貫性
- [ ] All screen titles use same text style (H1) | すべての画面タイトルが同じテキストスタイル（H1）を使用
- [ ] Section headers consistent (H2) | セクションヘッダーが一貫している（H2）
- [ ] Body text same size across screens | 本文テキストが画面間で同じサイズ
- [ ] Button text uses same style | ボタンテキストが同じスタイルを使用
- [ ] Text alignment consistent | テキスト整列が一貫している

**2. Spacing Consistency** | スペーシングの一貫性
- [ ] Screen margins same on all screens (16-24px) | 画面マージンがすべての画面で同じ（16-24px）
- [ ] Component spacing uniform (8px, 16px, 24px) | コンポーネントスペーシングが均一（8px、16px、24px）
- [ ] Padding inside cards/containers consistent | カード/コンテナ内のパディングが一貫している
- [ ] Navigation bar heights match | ナビゲーションバーの高さが一致
- [ ] List item spacing uniform | リストアイテムスペーシングが均一

**3. Component Consistency** | コンポーネントの一貫性
- [ ] All primary buttons same size and style | すべてのプライマリボタンが同じサイズとスタイル
- [ ] Input fields uniform height (48-56px) | 入力フィールドの高さが均一（48-56px）
- [ ] Icons same size where appropriate (24x24px) | 適切な場合にアイコンが同じサイズ（24x24px）
- [ ] Cards have consistent corner radius | カードが一貫したコーナー半径を持つ
- [ ] Navigation patterns match across screens | ナビゲーションパターンが画面間で一致

**4. Mobile Guidelines Adherence** | モバイルガイドラインの遵守
- [ ] All touch targets minimum 44x44px | すべてのタッチターゲットが最小44x44px
- [ ] Primary actions in easy-to-reach zones | 到達しやすいゾーンにプライマリアクション
- [ ] Text readable (minimum 16px for body) | テキストが読める（本文は最小16px）
- [ ] Content within safe areas (no notch overlap) | セーフエリア内のコンテンツ（ノッチオーバーラップなし）
- [ ] Bottom navigation clear of home indicator | ホームインジケーターから明確な下部ナビゲーション

**5. Visual Hierarchy** | ビジュアル階層
- [ ] Most important elements are most prominent | 最も重要な要素が最も目立つ
- [ ] Clear distinction between primary and secondary actions | プライマリアクションとセカンダリアクションの明確な区別
- [ ] Proper use of white space | 余白の適切な使用
- [ ] Content organized in logical sections | 論理的なセクションに整理されたコンテンツ
- [ ] Visual flow guides user naturally | ビジュアルフローがユーザーを自然にガイド

**6. Content Completeness** | コンテンツの完全性
- [ ] All interactive elements labeled | すべてのインタラクティブ要素がラベル付けされている
- [ ] Navigation clearly indicated | ナビゲーションが明確に示されている
- [ ] Error states considered | エラー状態が考慮されている
- [ ] Empty states shown where appropriate | 適切な場合に空の状態が表示される
- [ ] Loading states indicated | ローディング状態が示されている

**Refinement Process:**
1. **Global alignment check** | グローバル整列チェック
   - Select all elements on a screen | 画面上のすべての要素を選択
   - Use alignment tools to fix misalignments | 整列ツールを使用してミスアラインメントを修正
   - Check grid alignment | グリッド整列をチェック

2. **Spacing audit** | スペーシング監査
   - Measure distances between elements | 要素間の距離を測定
   - Adjust to match spacing system (4px, 8px, 16px, etc.) | スペーシングシステムに一致するように調整（4px、8px、16px など）
   - Use Figma's space-between tools or XD's padding | Figmaのspace-betweenツールまたはXDのパディングを使用

3. **Component update** | コンポーネント更新
   - Replace inconsistent elements with components | 一貫性のない要素をコンポーネントで置き換える
   - Update component master to fix all instances | すべてのインスタンスを修正するためにコンポーネントマスターを更新
   - Ensure all buttons, inputs, etc. use same components | すべてのボタン、入力などが同じコンポーネントを使用することを確認

4. **Cross-screen comparison** | 画面間比較
   - View all screens at once (zoom out) | すべての画面を一度に表示（ズームアウト）
   - Check that similar elements look similar | 類似の要素が類似に見えるかをチェック
   - Verify navigation consistency | ナビゲーションの一貫性を検証

**Bot Guidance:**

- Highlights inconsistencies: "Button on Screen 3 is 42px tall, others are 48px" | 一貫性のなさを強調：「画面3のボタンは42px高、他は48px」
- Provides checklist for manual review | 手動レビュー用チェックリストを提供

**Expected Output:** Refined wireframes with consistent design system applied across all screens

## Common Issues & Solutions | 一般的な問題と解決策

**Problem 1: Tool overwhelming - too many features | ツールが圧倒的 - 機能が多すぎる**
- **Solution:** Focus only on essential tools for this week: Frame/Artboard, Rectangle, Text, Selection. Ignore advanced features like prototyping, plugins, or complex effects. You only need to create static wireframes. Bot provides beginner-focused tutorial ignoring advanced features. Practice with simple shapes first before creating actual screens. | この週に必要不可欠なツールのみに焦点を当てる：フレーム/アートボード、長方形、テキスト、選択。プロトタイピング、プラグイン、または複雑なエフェクトなどの高度な機能は無視。静的ワイヤーフレームを作成するだけで良い。ボットは高度な機能を無視した初心者向けチュートリアルを提供。実際の画面を作成する前に単純な形状で練習。
- **Bot Assistance:** Bot provides simplified tutorial focusing on wireframing essentials only | ボットはワイヤーフレーミングの必需品のみに焦点を当てた簡略化されたチュートリアルを提供

**Problem 2: Wireframes too detailed/high-fidelity | ワイヤーフレームが詳細すぎる/高忠実度**
- **Solution:** Remember this is mid-fidelity wireframing, not final visual design. Keep it grayscale, use placeholder text, don't add photos or colors yet. Focus on layout and structure. Use simple rectangles for images. No need for pixel-perfect design - you'll refine next week. Save time by keeping it simple. | これは中忠実度のワイヤーフレーミングであり、最終的なビジュアルデザインではないことを覚えておく。グレースケールを保ち、プレースホルダーテキストを使用し、まだ写真や色を追加しない。レイアウトと構造に焦点を当てる。画像には単純な長方形を使用。ピクセル完璧なデザインは不要 - 来週洗練する。シンプルに保つことで時間を節約。
- **Bot Assistance:** Bot reminds about appropriate fidelity level and provides examples | ボットは適切な忠実度レベルについて思い出させ例を提供

**Problem 3: Inconsistent spacing and alignment | 一貫性のないスペーシングと整列**
- **Solution:** Always enable grid (Shift+G in Figma). Use alignment tools frequently (Align Left, Center, Distribute spacing). Create components for repeated elements so changes propagate. Measure spacing - don't eyeball it. Use 8px multiples. Review all screens side-by-side to spot inconsistencies. | 常にグリッドを有効にする（Figmaでは Shift+G）。整列ツールを頻繁に使用（左揃え、中央、スペーシングを分布）。繰り返し要素のコンポーネントを作成して変更が伝播するようにする。スペーシングを測定 - 目測しない。8pxの倍数を使用。すべての画面を並べてレビューして一貫性のなさを発見。
- **Bot Assistance:** Bot runs consistency checker and highlights spacing/alignment violations | ボットは一貫性チェッカーを実行し、スペーシング/整列違反を強調

**Problem 4: Components not working properly | コンポーネントが適切に機能しない**
- **Solution:** In Figma, select all elements that make up component, then Create Component (Cmd/Ctrl+Alt+K). To use, drag from Assets panel or Alt/Option-drag existing instance. To edit, find master component (click "Go to main component"). In XD, similar process with Make Component (Cmd/Ctrl+K). Watch bot's component tutorial video. | Figmaでは、コンポーネントを構成するすべての要素を選択し、次にコンポーネントを作成（Cmd/Ctrl+Alt+K）。使用するには、Assetsパネルからドラッグまたは既存のインスタンスをAlt/Optionドラッグ。編集するには、マスターコンポーネントを見つける（「メインコンポーネントに移動」をクリック）。XDでは、コンポーネントを作成（Cmd/Ctrl+K）で同様のプロセス。ボットのコンポーネントチュートリアルビデオを視聴。
- **Bot Assistance:** Bot provides step-by-step component creation and editing guide | ボットはステップバイステップのコンポーネント作成と編集ガイドを提供

**Problem 5: Touch targets still too small | タッチターゲットがまだ小さすぎる**
- **Solution:** Minimum 44x44px (Apple) or 48x48px (Material Design). In Figma/XD, select button and check dimensions in right panel. For text buttons, add invisible padding/frame to reach minimum size. Create component with proper size so all instances are correct. Use bot's touch target validator. | 最小44x44px（Apple）または48x48px（Material Design）。Figma/XDでは、ボタンを選択し右パネルで寸法をチェック。テキストボタンの場合、最小サイズに達するために見えないパディング/フレームを追加。適切なサイズでコンポーネントを作成してすべてのインスタンスが正しくなるように。ボットのタッチターゲットバリデーターを使用。
- **Bot Assistance:** Bot automatically checks all interactive elements and flags undersized targets | ボットは自動的にすべてのインタラクティブ要素をチェックし、サイズ不足のターゲットにフラグを立てる

**Problem 6: File management and organization messy | ファイル管理と整理が乱雑**
- **Solution:** Use clear naming: "Screen Name - State" (e.g., "Home - Default", "Search - Empty"). Group related elements in frames. Use layers panel to organize. Create separate pages/artboards for: Wireframes, Components, Archive. Name layers descriptively. Bot provides file organization templates. | 明確な命名を使用：「画面名 - 状態」（例：「ホーム - デフォルト」、「検索 - 空」）。関連要素をフレームにグループ化。レイヤーパネルを使用して整理。別のページ/アートボードを作成：ワイヤーフレーム、コンポーネント、アーカイブ。レイヤーを説明的に命名。ボットはファイル整理テンプレートを提供。
- **Bot Assistance:** Bot provides file organization best practices and naming conventions | ボットはファイル整理ベストプラクティスと命名規則を提供
