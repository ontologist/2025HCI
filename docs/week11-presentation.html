Analyzing Quantitative Data

定量的データの分析

1. Task Success Rate

Successful tasks / Total attempts × 100

Benchmark: >78% = Acceptable | <50% = Critical

2. Time on Task

Look for outliers and patterns

Fast ≠ always good (might skip steps)

3. Error Rate

Slips vs. Mistakes

Recovery time matters

Analyzing Qualitative Data

定性的データの分析

Quote Analysis

  • Direct user pain points
  • Expectations vs. reality
  • Group similar quotes by theme

Behavior Analysis

  • Hesitation points
  • Navigation paths
  • Repeated attempts
  • Body language

Pattern Rule: 3+ participants = Pattern worth addressing

Finding Patterns | パターンの発見

Strong Patterns (High Priority)

5/5 participants affected | Causes task failure

Moderate Patterns (Medium Priority)

3-4/5 participants affected | Causes delays

Weak Patterns (Low Priority)

1-2/5 participants affected | Minor impact

Triangulation: Combine quantitative + qualitative + behavioral data

Severity Rating Framework

Jakob Nielsen's 0-4 Scale

0 = Not a usability problem

1 = Cosmetic (fix if time allows)

2 = Minor usability problem (low priority)

3 = Major usability problem (high priority)

4 = Usability catastrophe (MUST FIX)

Prioritization Matrix

優先順位付けマトリックス

High Frequency Med Frequency Low Frequency
Critical Impact P1 P2 P3
Major Impact P2 P3 P4
Minor Impact P3 P4 P4

Frequency × Impact = Priority

Root Cause Analysis

根本原因分析

The "5 Whys" Technique

Symptom: Users can't complete checkout

Why? → Can't find button

Why? → Below the fold

Why? → Too much info above

Why? → Wanted single screen

Why? → Unvalidated assumption

Root Cause: Design decision based on unvalidated assumption

Solution: Test assumptions, prioritize critical actions

Actionable Recommendations

実行可能な推奨事項

❌ BAD

"Improve navigation"

"Make it user-friendly"

"Fix the issues"

✅ GOOD

"Move menu icon to top-left, increase to 44×44 px"

"Add price filter with '$' icon at top of list"

"Display errors in red with specific instructions"

Good recommendations are: Specific • Actionable • Evidence-based • Prioritized • User-centered

Recommendation Template

Issue: Price filter not visible

Priority: P1 (Critical)

Evidence: 4/5 users (80%) couldn't find it; 40% success rate; avg 4.2 min vs. expected 1.5 min

Root Cause: Hidden in collapsed section

Recommendations:

  1. Move to top of product list
  2. Display expanded by default
  3. Add clear "$" icon
  4. Increase height from 32px to 48px

Expected Impact: >78% success rate, ~1.5 min completion

Iterative Design Cycle

反復デザインサイクル

Design → Test → Analyze → Prioritize → Redesign → Test Again → Repeat

Why Iteration Matters:

  • First test reveals major issues
  • Second test reveals subtle issues
  • Each cycle improves design
  • "Fail fast, learn fast"

Remember: Design is never "done" – it continuously evolves

Common Mistakes to Avoid

避けるべき一般的なミス

  • ❌ Confirmation Bias: Only seeing positive data
  • ❌ Cherry-Picking: Ignoring negative findings
  • ❌ Ignoring Outliers: May reveal accessibility issues
  • ❌ Vague Recommendations: "Make it better"
  • ❌ Analysis Paralysis: Over-analyzing forever
  • ❌ Defending Design: If users don't understand, it's a design problem

Today's Activity

今日のアクティビティ (60 min)

Part 1: Testing (30 min)

  • Test with 2-3 participants
  • Use Week 10 test plan
  • Apply think-aloud protocol
  • Record all metrics
  • Take detailed notes

Part 2: Analysis (30 min)

  • Calculate metrics
  • Identify patterns
  • Apply severity ratings
  • Use priority matrix
  • Create 3-5 recommendations

Deliverable: Usability findings report with recommendations

What to Expect:

Note: Individual assessment – no external AI allowed

Key Takeaways | 重要なポイント

  1. Analysis transforms data into insights
    Combine quantitative + qualitative + behavioral
  2. Prioritization is essential
    Use severity ratings (0-4) and frequency × impact
  3. Recommendations must be actionable
    Specific, evidence-based, prioritized, user-centered
  4. Design is iterative
    Test → Analyze → Redesign → Test Again
