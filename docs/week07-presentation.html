Understanding User Flows

User Flow: The path users take to complete a task

Components:

  1. Entry Point: Where user starts
  2. Actions: What user does
  3. Decision Points: Choices user makes
  4. Exit Point: Where flow ends

Common Flows:

  • Onboarding: Welcome ‚Üí Login ‚Üí Tutorial ‚Üí Home
  • Search: Home ‚Üí Search ‚Üí Results ‚Üí Detail
  • Purchase: Browse ‚Üí Cart ‚Üí Checkout ‚Üí Confirmation

Creating Hotspots & Links

Hotspots: Clickable areas that trigger navigation

Types of Interactive Elements:

  • Buttons: Primary/secondary actions
  • Navigation: Tab bars, menus, back buttons
  • Components: Cards, list items, form fields

In Figma:

  1. Switch to Prototype mode
  2. Select element
  3. Drag connector to target screen
  4. Set trigger (On Tap)
  5. Choose transition

Navigation Principles

1. Clarity

Users always know where they are

  • Show current location
  • Clear page titles
  • Breadcrumbs for deep hierarchies

2. Consistency

Navigation in same place

  • Tab bars at bottom
  • Back buttons top-left
  • Same patterns throughout

3. Efficiency

Minimize taps to content

  • 3 taps or less
  • Shortcuts for frequent tasks
  • Quick access to home

4. Forgiveness

Allow users to go back

  • Always provide back button
  • Don't trap users
  • Offer cancel options

Building Multi-Screen Prototypes

Four-Step Process:

  1. Prepare Screens: Complete all wireframes, name descriptively
  2. Add Hotspots: Link buttons, connect tab bars, add back navigation
  3. Set Transitions: Choose appropriate animations, set duration (200-400ms)
  4. Test Flow: Click through, verify links, check navigation

Best Practices

‚úÖ DO's

  • Focus on 2-3 key flows
  • Start with basic navigation
  • Test early and often
  • Keep transitions consistent
  • Include back navigation

‚ùå DON'Ts

  • Add every possible detail
  • Link every element
  • Use random transitions
  • Forget mobile gestures
  • Skip self-testing

Common Mistakes

  1. Too Many Connections: Focus on 2-3 key flows
  2. Inconsistent Transitions: Use same for same actions
  3. Missing Back Navigation: Always include back button
  4. Over-Animation: Keep animations subtle
  5. Not Testing: Test every flow yourself first
