    Managing Presentations | ãƒ—ãƒ¬ã‚¼ãƒ³ãƒ†ãƒ¼ã‚·ãƒ§ãƒ³ã®ç®¡ç†

    Presentation Flow

    Before Each Presentation:

    1. Announce next presenter

    2. Give them 1-2 minutes to set up

    3. Help with technical issues if needed

    4. Distribute peer feedback forms

    5. Start timer when ready

    During Presentation:

    1. Monitor time (give 2-minute and 1-minute warnings)

    2. Take notes on strengths to mention

    3. Prepare thoughtful question if Q&A is slow

    4. Watch audience engagement

    5. Note any technical issues

    After Presentation:

    1. Lead applause

    2. Facilitate Q&A (2-3 minutes)

    3. Ask your own question if needed

    4. Give 2 minutes for peer feedback completion

    5. Thank presenter

    6. Transition to next presenter

    Time Management Strategies

    If Running Over:
    • Politely interrupt with time warnings
    • "That's our 10-minute mark, please wrap up"
    • Allow to finish thought but keep brief
    • Adjust Q&A time if needed
    If Running Short:
    • Extended Q&A
    • Ask follow-up questions
    • Encourage deeper discussion
    • Extra feedback time
    If Technical Issues:
    • Stay calm and supportive
    • "Let's take a minute to troubleshoot"
    • Have backup options ready
    • Don't count setup time against presentation

    Encouraging Good Questions

    Model Good Questions:
    • "Can you explain why you chose that color scheme?"
    • "How did user feedback influence that design decision?"
    • "What accessibility features did you include?"
    • "If you had more time, what would you add?"
    If No Questions:
    • Ask your prepared question
    • "I'm curious about..."
    • "I noticed... can you tell us more?"
    • Highlight something impressive and ask about it
    If Inappropriate Questions:
    • Redirect diplomatically
    • "Let's focus on the design decisions"
    • "That's more about personal preference than design principles"
    • Protect presenters from harsh criticism

    Using the Presentation Rubric

    Evaluation Categories:

    1. User Research (5 points)

    2. Design Process (5 points)

    3. Final Prototype (5 points)

    4. Usability Testing (5 points)

    5. Accessibility (5 points)

    6. Clarity and Organization (4 points)

    7. Demonstration (3 points)

    8. Reflection and Insights (3 points)

    Total: 35 points

    Evaluation Tips

    Be Fair and Consistent:
    • Use the same standards for all students
    • Compare to rubric, not to other students
    • Credit effort and growth
    • Consider starting point and improvement
    Look For:
    • Evidence of design thinking process
    • User-centered decisions
    • Clear rationale for choices
    • Thoughtful iteration
    • Professional execution
    • Connection to HCI principles
    Common Strengths to Recognize:
    • Thorough user research
    • Clear persona development
    • Multiple design iterations
    • Effective usability testing
    • Strong accessibility features
    • Professional presentation
    • Honest reflection
    Common Areas for Growth:
    • More user testing needed
    • Accessibility could be stronger
    • Design rationale unclear
    • Limited iterations shown
    • Time management in presentation

    Technical Difficulties

    Prototype Won't Load:
    • "That's okay, walk us through with screenshots"
    • "Describe what would happen if we could see it"
    • "Show us your design files instead"
    • Don't penalize for technical issues outside student control
    Connectivity Issues:
    • Have offline backup plan
    • Use personal hotspot if available
    • Continue with non-demo portions
    • Reschedule demo if necessary
    Device Compatibility:
    • Have multiple adapters ready
    • Test connections before class
    • Have backup laptop available
    • Allow presentation from instructor's computer

    Student Anxiety

    Very Nervous Presenters:
    • Smile and make eye contact
    • Nod encouragingly
    • Ask easy warm-up question
    • Highlight strengths during Q&A
    • Remind them it's okay to be nervous
    Presentation Freezing:
    • "Take a deep breath"
    • "Would you like some water?"
    • Ask specific question to restart
    • Offer to come back to them later if needed
    Time Running Out:
    • "You're doing great, let's move to the demo"
    • Help them prioritize remaining content
    • Allow to finish key points briefly

    Quality Issues

    Incomplete Work:
    • Focus on what was accomplished
    • Ask about challenges faced
    • Discuss lessons learned
    • Credit effort and partial completion
    • Mention growth and improvement
    Weak Presentation:
    • Ask probing questions to draw out understanding
    • Highlight any strengths
    • Provide constructive feedback privately later
    • Remember presentation skills â‰  design skills
    Exceptional Work:
    • Give deserved praise
    • Ask to share specific techniques
    • Use as learning example for class
    • Encourage portfolio development
    • Don't make others feel inadequate

    Last-Minute Issues

    Student Absent on Presentation Day:
    • Contact immediately
    • Offer makeup presentation
    • Accept video submission
    • Be flexible with valid excuses
    • Don't penalize for emergencies
    Multiple Technical Failures:
    • Move to backup classroom if possible
    • Allow screenshot presentations
    • Extend class time if permitted
    • Schedule continuation session
    • Don't count tech issues against students
    Student Unable to Present:
    • Offer alternative demonstration
    • Accept pre-recorded presentation
    • One-on-one presentation with instructor
    • Extended office hours presentation
    • Portfolio submission alternative
    Overwhelming Emotion:
    • Some students may cry from stress/relief
    • Have tissues available
    • Offer break if needed
    • Private conversation if appropriate
    • Normalize emotional response
