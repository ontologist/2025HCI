## Slide 6: Fidelity Levels Explained
**Title:** Fidelity Levels Explained | å¿ å®Ÿåº¦ãƒ¬ãƒ™ãƒ«ã®èª¬æ˜

**Three-Column Layout:**

**Low-Fidelity (Lo-Fi):**
- Simple shapes and boxes
- Grayscale or black and white
- Placeholder text (Lorem ipsum)
- Basic layout only

When to Use: Early design exploration, rapid iteration, testing multiple concepts

**Medium-Fidelity (Mid-Fi):**
- More detailed layout
- Actual content (not Lorem ipsum)
- Some visual design elements
- Limited color (brand colors)

When to Use: Presenting to stakeholders, content structure validation, early usability testing

**High-Fidelity (Hi-Fi):**
- Full visual design
- Real content and images
- Brand colors, fonts, icons
- Interactive elements

When to Use: Final usability testing, developer handoff, marketing materials, stakeholder approval

## Slide 15: Common Wireframing Mistakes
**Title:** Common Wireframing Mistakes | ä¸€èˆ¬çš„ãªãƒ¯ã‚¤ãƒ¤ãƒ¼ãƒ•ãƒ¬ãƒ¼ãƒŸãƒ³ã‚°ã®é–“é•ã„

**Mistakes to Avoid:**

**1. Too Much Detail Too Soon:**
- Problem: Adding colors, images, fonts in wireframes
- Solution: Stay in grayscale, use placeholders
- Why: Focus on structure first

**2. Inconsistent Spacing:**
- Problem: Random gaps between elements
- Solution: Use a grid system, consistent margins
- Why: Creates visual hierarchy and clarity

**3. Ignoring Mobile Constraints:**
- Problem: Desktop-like layouts on mobile
- Solution: Design for mobile screen sizes first
- Why: Mobile is the primary use case

**4. No Content Priority:**
- Problem: Everything looks equally important
- Solution: Use size, position, contrast for hierarchy
- Why: Users need to know what's most important

**5. Forgetting Navigation:**
- Problem: No way to move between screens
- Solution: Include navigation elements in every screen
- Why: Users need clear paths through the app

**Visual Elements:**
- Before/after examples of each mistake
- Warning icons
- Corrected examples
- Best practice reminders
