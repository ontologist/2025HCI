## Slide 7: Transition Types
**Title:** Transition Types | ãƒˆãƒ©ãƒ³ã‚¸ã‚·ãƒ§ãƒ³ã‚¿ã‚¤ãƒ—

**Five Main Types:**

**1. Instant**
- No animation
- Use for: Quick actions, tab switching

**2. Dissolve/Fade**
- Crossfade between screens
- Use for: Modal overlays, subtle changes

**3. Slide**
- Screens slide from direction
- Use for: Sequential flows, hierarchical navigation

**4. Push**
- Both screens move together
- Use for: Deep navigation hierarchies

**5. Modal**
- Slides up from bottom
- Use for: Temporary tasks, interruptions

## Slide 9: Designing Effective Navigation
**Title:** Designing Effective Navigation | åŠ¹æœçš„ãªãƒŠãƒ“ã‚²ãƒ¼ã‚·ãƒ§ãƒ³ã®ãƒ‡ã‚¶ã‚¤ãƒ³

**Four Key Principles:**

**1. Clarity**
- Users always know where they are
- Show current location in navigation
- Use clear page titles and headings

**2. Consistency**
- Navigation elements in same place
- Tab bars always at bottom
- Back buttons always top-left

**3. Efficiency**
- Minimize taps to reach content
- Aim for 3 taps or less to any screen
- Provide shortcuts for frequent tasks

**4. Forgiveness**
- Allow users to go back
- Always provide back button or gesture
- Don't trap users in flows

- Navigation examples (good vs bad)

## Slide 15: Common Mistakes
**Title:** Common Prototyping Mistakes | ä¸€èˆ¬çš„ãªãƒ—ãƒ­ãƒˆã‚¿ã‚¤ãƒ”ãƒ³ã‚°ã®é–“é•ã„

**Mistakes to Avoid:**

**1. Too Many Connections**
- Problem: Linking every possible element
- Solution: Focus on 2-3 key user flows
- Why: Overwhelming, hard to maintain

**2. Inconsistent Transitions**
- Problem: Random animations for similar actions
- Solution: Use same transition for same action type
- Why: Creates confusion

**3. Missing Back Navigation**
- Problem: No way to return to previous screen
- Solution: Always include back button
- Why: Users get trapped

**4. Over-Animation**
- Problem: Too many fancy transitions
- Solution: Keep animations subtle
- Why: Slows down interaction

**5. Not Testing**
- Problem: Assuming navigation works
- Solution: Test every flow yourself
- Why: Broken links, dead ends

- Before/after mistake corrections
- Warning icons
- Best practice reminders
