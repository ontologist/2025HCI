## Slide 6: Presentation Rubric

**Design Quality (15 points)**

## Slide 8: Peer Feedback Guidelines
**Title:** Peer Feedback Guidelines | ピアフィードバックガイドライン

**Two-Column Layout:**

**DO: ✅**
- Be Specific: "The color contrast improves readability"
- Focus on Design: Not personal preferences
- Use HCI Principles: Reference course concepts
- Balance Positive and Constructive
- Be Respectful and Supportive

**DON'T: ❌**
- Don't Be Vague: Avoid "It's fine"
- Don't Make It Personal: Focus on design
- Don't Only Criticize: Include strengths
- Don't Suggest Complete Redesigns
- Don't Compare to Other Students

**Visual Elements:**
- Checkmarks and X marks
- Good vs. bad feedback examples
- Color coding (green for DO, red for DON'T)
- Example quotes

## PowerPoint Design Tips:

### Visual Design:
- Use celebratory, warm color scheme (golds, warm blues, greens)
- Include plenty of white space
- Use large, readable fonts
- Add student achievement graphics
- Include bilingual text clearly
- Use congratulatory imagery

### Layout:
- Keep slides uncluttered but celebratory
- Use bullet points effectively
- Include visual breaks between sections
- Maintain consistent formatting
- Add divider slides between major sections

### Interactive Elements:
- Add animations for celebration elements
- Use transitions between slides
- Include timers for presentations
- Embed student work examples (with permission)
- Prepare for Q&A discussions

### Accessibility:
- Use high contrast colors
- Include alt text for images
- Ensure readable font sizes
- Consider colorblind-friendly palettes
- Provide captions if using videos

### Celebration Elements:
- Achievement badges
- Progress charts
- Before/after comparisons
- Success stories
- Student highlights (with permission)
- Portfolio showcase examples
- Congratulatory messages
- Course journey visualization
